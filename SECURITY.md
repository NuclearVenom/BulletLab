# Security Policy

## Supported Versions

We take the security of BulletLab seriously. The following table outlines the versions that currently receive security updates.

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |
| Future versions | :white_check_mark: |
| < 0.1.0 | :x:                |

## Reporting a Vulnerability

**Please do NOT report security vulnerabilities as public GitHub issues.**

If you discover a security vulnerability in BulletLab, please practice responsible disclosure. We appreciate your efforts to keep our community safe.

To report a vulnerability:
1. Contact the maintainer directly or use GitHub's private vulnerability reporting feature (if enabled on the repository).
2. Provide a detailed description of the vulnerability, including steps to reproduce it.
3. Allow us a reasonable amount of time to review, patch, and release an update before disclosing the vulnerability publicly.

We will acknowledge receipt of your report promptly and keep you informed of our progress in addressing the issue.
