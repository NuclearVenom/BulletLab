# Support

Need help with BulletLab? We're here to help! Below are the best ways to get assistance, report issues, or suggest improvements.

## Where to ask questions

### Documentation
Your first stop should be the [BulletLab Documentation](https://nuclearvenom.github.io/BulletLab/). It contains:
- Getting Started Guides
- Core Concepts
- Detailed API Reference
- A Cookbook with common snippets and patterns

### GitHub Discussions
If you have a question about how to use BulletLab, need architecture advice, or want to share something you've built, please use **[GitHub Discussions](https://github.com/NuclearVenom/BulletLab/discussions)** (if enabled). This is the best place for open-ended questions.

## Feature Requests
If you have an idea for a new feature or improvement:
1. Check the [Roadmap](ROADMAP.md) to see if it's already planned.
2. Search existing [Issues](https://github.com/NuclearVenom/BulletLab/issues) to ensure it hasn't already been requested.
3. Open a new **Feature Request Issue**. Describe the feature, why it is needed, and how it aligns with BulletLab's goals.

## Bug Reports
If you encounter unexpected behavior or errors, please open a **[Bug Report Issue](https://github.com/NuclearVenom/BulletLab/issues)**.

### What to include in a bug report:
To help us resolve issues quickly, please provide:
- **BulletLab version** (`pip show bulletlab`)
- **Python version**
- **Operating system**
- A **minimal, reproducible code example** that demonstrates the problem.
- The **expected behavior** and the **actual behavior**.
- Full error tracebacks (if applicable).
